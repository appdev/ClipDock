# Delivery Workflow

日期：2026-05-08

执行者：Codex

## 版权与原创边界

本项目可以参考 Paste 的产品类别、macOS 系统质感和剪贴板管理器常见交互，但不做 Paste 的像素级复刻，不复制 Paste 的品牌、文案、专有布局组合或可识别视觉表达。所有设计交付必须是原创方案。

## 角色与阶段门

1. UI 设计师产出 `docs/ui-design.md`。
2. QA 审查 UI 是否满足“Paste 启发但原创”的目标，结论写入 `docs/ui-qa-review.md`。
3. 只有 UI QA 通过后，架构师基于 UI 文档产出 `docs/architecture.md`。
4. QA、UI、开发共同评审架构，结论写入 `docs/architecture-review.md`。
5. 只有架构评审通过后，开发按功能切片实现。
6. 每完成一个完整功能，QA 必须验收并在 `docs/feature-qa-log.md` 记录，通过后再进入下一功能。

## 功能切片顺序

1. 面板视觉与基础布局。
2. 用户设置窗口。
3. 剪贴板历史数据模型与本地存储。
4. 剪贴板捕获与来源应用信息。
5. 搜索、筛选与键盘操作。
6. 粘贴写回与自写入抑制。
7. 偏好设置持久化。
8. 文件剪贴板捕获。
9. 忽略列表持久化与捕获跳过规则。
10. 窗口标题采集与标题关键词运行时规则。
11. 本地数据维护。
12. GUI 回归测试地基。
13. 截图级 GUI 回归雏形。
14. 主面板 UI 精简。
15. Login Item 启动时运行。
16. 条目管理。
17. 主面板性能优化。
18. 系统权限状态提示。
19. 真实设备 UI QA 探针。
20. 图片预览后台加载。
21. 图标缓存维护。

冻结范围：同步、导入和导出暂不进入近期功能切片；文件捕获、忽略列表、窗口标题采集、本地数据维护、GUI 回归测试地基、截图级 GUI 回归雏形、主面板 UI 精简、Login Item 启动时运行、条目管理、主面板性能优化、系统权限状态提示、真实设备 UI QA 探针、图片预览后台加载和图标缓存维护已完成，下一阶段只能选择真实窗口交互自动化、产品化打包或其他非同步/非导出方向。

## 验收标准

- 本地 `swift build` 必须通过。
- 新功能必须有清晰的人工可观察行为和自动可执行验证。
- 文档必须记录日期、执行者、结论和遗留风险。
- 任何 UI 设计必须保持原创，不能以像素复刻 Paste 为目标。
