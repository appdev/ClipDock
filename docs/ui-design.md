# macOS 剪贴板管理器 UI 设计方案

日期：2026-05-08  
执行者：Codex UI Designer

## 1. 设计定位

本方案为一个 macOS 剪贴板管理器提供 Paste 启发但原创的界面方向。它保留系统级剪贴板管理器常见范式：快速呼出、横向浏览、搜索过滤、来源应用识别、固定收藏、偏好设置与状态反馈；同时避免 Paste 的像素级克隆、品牌词、文案、图标组合、独特卡片比例和可识别视觉编排。

设计目标：

- 快速：从呼出到定位条目，默认路径以鼠标浏览和双击取用为主，键盘只作为辅助加速。
- 克制：使用 macOS vibrancy、系统字体、轻量层级，不制造厚重控制台感。
- 可读：文本、链接、图片、文件、颜色等内容有明确区分，但不依赖花哨装饰。
- 原创：以“底部全宽工作台 + 轻量顶部工具条 + 条目内容带 + 临时预览浮层”的结构表达剪贴板历史，不复刻 Paste 的视觉节奏。

## 2. 视觉原则

整体气质为“半透明工具面板”，像 Spotlight 与 Finder 预览的混合体。界面应像系统能力的自然延伸，而不是品牌化强烈的独立产品。

原创边界：

- 不使用 Paste 名称、品牌色、品牌文案或可识别营销表达。
- 不复刻 Paste 的卡片尺寸、横向堆叠比例、动画曲线和内容布局组合。
- 可使用 macOS 标准材料效果、SF 字体、系统图标语义和剪贴板管理器常规功能。
- 视觉识别点采用“贴底全宽面板 + 顶部高度调节横条 + 横向内容带 + 轻量预览浮层”，形成不同于 Paste 的信息结构。

## 3. 应用主体面板

主体面板默认通过菜单栏图标或快捷键呼出，固定在鼠标所在显示器底部。它像当前 Swift/AppKit demo 一样使用整块底部工作区：宽度等于当前显示器完整宽度，底边贴齐显示器底边，并覆盖 Dock 所在区域。

尺寸：

- 宽度：始终等于当前鼠标所在 `NSScreen.frame.width`，禁止用户调宽。
- 高度：默认约 320 px。
- 最小高度：260 px，保证搜索、筛选与至少一行条目可用。
- 最大高度：560 px，或当前显示器高度的 62%，取较小值。
- 位置：`x = screen.frame.minX`，`y = screen.frame.minY`，不要使用会避开 Dock 的 `visibleFrame` 作为底边依据。
- 圆角：顶部左右 16 px，底部左右 0 px，底边与屏幕边缘贴合。
- 阴影：仅顶部投影，0 -16 px 44 px rgba(0, 0, 0, 0.22)，暗色模式降低到 0.16。
- 边框：顶部 1 px rgba(255, 255, 255, 0.22)，暗色模式使用 rgba(255, 255, 255, 0.12)。
- 调整方式：只允许拖动顶部横条调整高度；窗口侧边、底边和角点不提供宽度或自由缩放手柄。

布局：

- 顶部高度调节横条：高度 10 px，hover 时显示上下调整光标。
- 顶部轻量工具条：高度 54 px。
- 条目内容带：占据剩余高度，横向滚动或按可用宽度显示更多条目。
- 面板内边距：16 px。
- 区块间距：12 px。
- 面板宽度变化仅来自当前显示器宽度变化或鼠标切换显示器，不能来自用户拖拽。

材料：

- 主窗口背景：macOS `NSVisualEffectView`，material 建议使用 `.hudWindow` 或 `.popover`。
- 顶部工具条背景：不再使用厚重搜索栏底板，保持半透明留白，只用按钮和 chip 形成操作入口。
- 条目内容带不使用嵌套卡片墙；通过固定条目单元、分隔线、间距和背景层级区分。

## 4. 顶部搜索与筛选

顶部只保留轻量搜索入口和类型 chip。默认状态不展示大搜索框，避免底部面板第一眼被控件占满；点击搜索图标或按 `Command + F` 后，搜索框在图标右侧展开并获得焦点。

搜索入口规格：

- 搜索按钮：30 x 30 px，圆形半透明背景，使用系统放大镜图标。
- 展开搜索框：宽度约 220 px，高度 30 px。
- 占位文案：`搜索剪贴板内容、应用或类型`。
- 字体：SF Pro Text 13。
- 背景：沿用系统 `NSSearchField`，不再额外包裹大面积卡片底板。
- 收起规则：搜索为空时再次点击搜索图标可收起；有关键词时保留输入状态。

类型 chip：

- 选项：`剪贴板`、`文本`、`链接`、`图片`、`文件`。
- 高度：30 px，圆角 15 px。
- 每个类型 chip 使用 1 个小色点和短标签，`剪贴板` 表示全部历史。
- 当前选中项使用更实的半透明白底和强调色边界，不使用 segmented control。
- chip 只负责类型筛选，不承担来源应用筛选、固定、清理或更多菜单。

键盘行为：

- `Command + F` 聚焦搜索框。
- `Escape` 清空搜索；搜索为空时关闭面板。
- `Command + 1...5` 快速选中当前可见的第 1 到第 5 个条目。
- 左右方向键移动条目选择。
- 单击条目选中，双击条目复制内容到系统剪贴板并隐藏面板。
- `Space` 展开或收起临时预览浮层。

## 5. 顶部轻量工具条

主体面板不设置常驻左侧轨道。筛选能力集中在顶部横条中，避免在底部面板里占用垂直浏览空间。

分组：

- 剪贴板：默认入口，展示全部历史。
- 类型：文本、链接、图片、文件。
- 搜索：通过搜索图标临时展开。

视觉规格：

- 类型筛选使用 chip，放在搜索入口右侧。
- 主面板顶部不常驻来源应用图标组，不放更多菜单、关闭按钮、固定入口或清理建议入口。
- 来源应用图标只在条目卡片内作为来源线索展示；来源聚合能力保留在数据层，除非后续有明确需求，不重新放回主面板顶部。
- 横条高度：54 px，含搜索框、筛选控件和操作按钮。
- 图标：16 px，居中对齐。
- 控件圆角：圆形按钮 15 px，chip 15 px。
- 当前项背景：系统强调色透明度 14%。
- 悬停背景：当前文字色透明度 8%。

交互：

- 点击筛选项立即更新条目内容带。
- 搜索框展开后输入会实时刷新条目内容带。
- 主体面板始终保持当前显示器全宽，不出现低宽度侧栏折叠逻辑。

## 6. 剪贴板条目列表

条目区域位于底部面板主体内容带内。默认采用横向滚动的固定单元，而不是 Paste 式卡片比例或品牌化堆叠节奏；每个单元是可扫读的内容摘要，面板越宽显示的条目越多。

条目尺寸：

- 默认单元宽度：218 px。
- 默认单元高度：跟随面板内容区高度，默认约 220 px。
- 紧凑模式单元宽度：176 px。
- 图片缩略图区域：72...172 px，随面板高度增长。
- 单元圆角：10 px。
- 单元内边距：12 px。
- 单元间距：8 px。
- 横向滚动边缘留白：16 px。

条目结构：

- 顶部色条：类型、复制时间和来源应用图标；颜色优先由来源 App / collection 稳定映射，同一来源保持同色，缺少来源时才回退内容类型色。
- 中部：内容摘要或缩略图，文本允许多行展示。
- 底部行：简短内容信息和类型文字。
- 选中态显示临时预览浮层，不在主面板内开辟常驻右侧详情栏。
- 鼠标滚轮纵向滚动必须映射为横向浏览；触控板横向滚动保持系统手感。
- 搜索框输入时实时刷新结果；类型 chip 立即过滤当前历史。
- 键盘支持 `Command + F` 聚焦搜索、`Command + 1...9` 直接复制当前完整可见条目、左右方向键移动选中项、`Space` 开关预览、`Escape` 清空搜索或隐藏面板；鼠标单击选中条目，双击复制到剪贴板并隐藏面板。

内容摘要规则：

- 文本：显示前 3 行，保留换行暗示，超出使用尾部省略。
- 链接：主行显示页面标题或域名，副行显示完整 URL 的中段省略。
- 图片：显示适配单元宽度的缩略图，保留 4 px 圆角，旁边或下方显示尺寸与文件大小。
- 文件：显示文件名、扩展名、路径末端 2 级。
- 颜色：显示 28 x 28 px 色块，文字显示 HEX、RGB 或系统色名。
- 富文本：显示纯文本摘要，并在副行标记“保留格式”。

状态样式：

- 默认：透明背景。
- 悬停：rgba(255, 255, 255, 0.34)；暗色 rgba(255, 255, 255, 0.07)。
- 选中：卡片边界使用系统强调色，顶部色条仍保留来源 App / collection 色。
- 已固定：当前主面板不展示固定入口或 pin 图标，避免无关元素占用卡片空间。
- 已忽略来源：命中应用标识或未知来源跳过规则的内容不进入列表，仅在设置窗口中可恢复规则。

## 7. 来源应用图标

来源应用图标是识别条目的核心线索，但不做品牌化装饰。

规格：

- 标准尺寸：32 x 32 px。
- 大预览尺寸：40 x 40 px。
- 圆角：遵循应用原始图标，不额外裁切为统一胶囊形。
- 缺失图标：使用系统默认应用占位图标，叠加 1 字母来源简称。

展示规则：

- 图标右下角可叠加 12 px 类型徽标，例如文本、链接、图片、文件。
- 徽标背景使用窗口背景的反向半透明层，确保亮暗模式可读。
- 同一应用连续多条记录不合并，但可在副信息中显示“同来源连续复制”。

## 8. 临时详情预览

详情预览用于确认内容，避免误粘贴。它不是常驻右侧检查器，也不改变主面板宽度；默认从当前选中条目上方弹出轻量浮层，必要时也可在条目单元内临时展开。

布局：

- 顶部：来源应用、复制时间和必要状态；固定、更多等低频操作不在主面板常驻展示。
- 中部：内容预览区。
- 底部：操作栏。

尺寸与间距：

- 浮层宽度：360 px 到 480 px，依据内容类型和当前显示器剩余空间自适应。
- 浮层最大高度：360 px，不超过主面板上方可见空间。
- 浮层锚点：当前选中条目的顶部中心，超出屏幕时向内侧偏移。
- 内边距：16 px。
- 顶部元信息高度：44 px。
- 操作栏高度：44 px。
- 内容区与操作栏间距：12 px。
- 主面板高度低于 260 px 时，预览优先使用面板上方浮层，不在内容带内展开。

操作：

- 复制到剪贴板：主按钮；双击条目执行相同动作并隐藏面板。
- 复制：图标按钮。
- 固定：放入条目右键管理菜单，不占用主面板卡片空间。
- 删除：放入条目右键管理菜单；删除后从当前列表移除。
- 打开来源应用：当来源可用时显示。

预览细节：

- 文本预览使用等宽字体开关；默认 SF Pro Text，代码片段自动切换 SF Mono。
- 链接预览显示域名、路径、标题和 favicon。
- 图片预览适应容器，保留透明棋盘格背景。
- 文件预览显示文件图标、名称、路径、大小和最后修改时间。
- 颜色预览显示大色块、HEX、RGB、HSL，并提供复制格式菜单。

## 9. 用户设置窗口 / 偏好设置

设置窗口采用 macOS 26 风格独立窗口，不放在主体浮层内；视觉参考系统设置的圆角侧栏、轻毛玻璃背景和分组卡片行。

窗口规格：

- 默认尺寸：920 x 700 px。
- 最小尺寸：820 x 600 px。
- 标题栏透明并使用 full-size content view。
- 外层窗口圆角：24 px。
- 左侧设置导航宽度：264 px，内边距 16 px，顶部留白 74 px。
- 内容区内边距：36 px，内容最大宽度 640 px。

设置分区：

- 通用：启动时运行、菜单栏显示、底部面板默认高度。
- 隐私：不记录的应用、窗口标题关键词、未知来源跳过；页内展示窗口标题采集所需的辅助功能权限状态，并提供系统设置入口。
- 键盘快捷键：打开剪贴板、`Command + 1...9` 快速取用、搜索、预览。
- 保留历史：保存数量、保留时长、是否记录图片和文件。
- 外观：亮色、暗色、跟随系统；条目密度；预览浮层开关。
- 当前不展示同步、导入或导出入口；本地数据维护类操作必须等对应 Rust mutation 完成后再加入 UI。

控件规范：

- 二元项使用 macOS switch。
- 数量或时长使用 stepper + 文本输入。
- 模式使用 segmented control。
- 快捷键使用右侧圆角 key pill，只展示当前固定快捷键，不伪造未实现的快捷键编辑器。
- 每个设置大组使用 18 px 圆角卡片，行高不低于 62 px，卡片内行之间使用 1 px 内缩分隔线。
- 未来若加入本地数据维护危险操作，应放在页面底部独立区域，按钮使用系统破坏性样式。

设置文案：

- 文案必须描述结果，不使用夸张品牌语气。
- 每个设置项标题不超过 16 个中文字符。
- 说明文字不超过 28 个中文字符，必要时放入帮助按钮。

## 10. 状态、空态与错误态

加载态：

- 面板首次打开时显示 3 个条目单元骨架屏。
- 骨架屏尺寸与真实条目单元一致，避免布局跳动。
- 加载超过 800 ms 后显示轻量进度指示。

空态：

- 无历史记录：显示系统剪贴板图标、标题 `暂无剪贴板记录`、说明 `复制内容后会显示在这里`。
- 搜索无结果：显示搜索图标、标题 `没有匹配内容`、说明 `尝试更短的关键词或切换筛选`。
- 筛选为空：显示对应类型图标、标题 `此类型暂无记录`。

错误态：

- 读取失败：在条目位置显示内联错误行，提供重试按钮。
- 权限不足：显示系统权限图标，说明需要在系统设置中允许对应系统能力。
- 来源文件丢失：文件条目保留历史摘要，并标记 `原文件不可用`。
- 图片过大：显示缩略占位和文件信息，预览浮层提示 `预览已跳过`。

错误态原则：

- 不使用大面积红色背景。
- 主错误文字使用系统红色，辅助说明使用次级文字。
- 可恢复错误必须提供明确操作。
- 不可恢复错误保留可复制的摘要信息。

## 11. 颜色

颜色使用系统语义色优先，减少自定义色值。

亮色模式：

- 主背景：vibrancy material。
- 主文字：rgba(0, 0, 0, 0.86)。
- 次级文字：rgba(0, 0, 0, 0.54)。
- 分隔线：rgba(0, 0, 0, 0.10)。
- 悬停层：rgba(0, 0, 0, 0.05)。
- 选中层：系统强调色 18%。

暗色模式：

- 主背景：vibrancy material。
- 主文字：rgba(255, 255, 255, 0.88)。
- 次级文字：rgba(255, 255, 255, 0.58)。
- 分隔线：rgba(255, 255, 255, 0.12)。
- 悬停层：rgba(255, 255, 255, 0.07)。
- 选中层：系统强调色 22%。

功能色：

- 成功：系统绿色。
- 警告：系统黄色。
- 错误：系统红色。
- 信息：系统蓝色。
- 固定：系统强调色。

避免项：

- 避免大面积紫蓝渐变、霓虹边框和品牌化高饱和背景。
- 避免把每种内容类型设计成强烈彩色卡片。
- 避免用颜色作为唯一状态区分，必须配合图标或文字。

## 12. 字体

字体使用 macOS 系统字体。

- 主字体：SF Pro Text。
- 标题：SF Pro Display，仅用于设置窗口页面标题或空态标题。
- 等宽：SF Mono，用于代码、终端片段、纯文本模式。

字号：

- 面板标题或页面标题：20 px / 26 px。
- 搜索输入：14 px / 20 px。
- 条目主文本：14 px / 19 px。
- 条目副文本：12 px / 16 px。
- 轨道标题：11 px / 14 px。
- 按钮文字：13 px / 18 px。
- 预览正文：13 px / 20 px。

字体权重：

- 页面标题：600。
- 条目主文本：500。
- 普通正文：400。
- 次级说明：400。

## 13. 间距与网格

基础间距单位为 4 px。

- 极小间距：4 px。
- 小间距：8 px。
- 标准间距：12 px。
- 区块间距：16 px。
- 页面边距：24 px。
- 设置页大区块间距：28 px。

对齐：

- 主面板顶部横条、搜索区和条目内容带均对齐 4 px 网格。
- 条目图标、标题、元信息基线保持一致。
- 操作按钮使用 32 x 32 px 点击区域。
- 所有 icon-only 按钮必须有 tooltip。

## 14. 动效与交互反馈

动效应轻、短、可预测。

- 面板呼出：从屏幕底部向上 18 px 进入，opacity 0 到 1，时长 140 ms。
- 面板关闭：向屏幕底部回落 12 px，opacity 1 到 0，时长 100 ms。
- 条目选择：背景色过渡 90 ms。
- 预览浮层切换：内容淡入 120 ms，不横向滑动整面板。
- 条目取用成功：卡片选中态短暂保持，面板随后隐藏。

反馈：

- 粘贴成功后面板立即关闭。
- 复制到剪贴板时，双击条目立即写回系统剪贴板并隐藏面板。
- 删除条目或清空当前结果后，当前列表立即刷新；固定条目不受批量清空影响。
- 搜索输入时列表实时过滤，延迟不超过 80 ms。

## 15. 可访问性

- 所有文字对比度满足 WCAG AA。
- 支持 VoiceOver 朗读条目类型、来源应用、复制时间和当前选中状态。
- 支持键盘完整操作，不要求鼠标。
- 支持减少动态效果；开启后禁用缩放，仅保留淡入淡出。
- 支持增大文字时提高建议面板高度，仍保持底部全宽、只允许调高度；条目单元随内容扩展。

## 16. 响应式与多屏

- 面板每次呼出都根据当前鼠标位置选择显示器，固定贴在该显示器底部。
- 多显示器场景下，鼠标移动到另一块屏幕后再次呼出，面板立即跟随到新的显示器。
- 面板宽度始终等于当前显示器完整宽度，不设置低宽度断点，不提供侧栏折叠方案。
- 面板底边使用 `NSScreen.frame.minY`，覆盖 Dock 区域；不要使用 `visibleFrame.minY` 留出 Dock 空隙。
- 显示器宽度较窄时，顶部仍只保留搜索入口和类型 chip，条目单元尺寸保持稳定。
- 高度由顶部横条调整，默认约 320 px；每块显示器可记住上次高度，但必须受最小高度和最大高度限制。
- 外接显示器分辨率或缩放变化后，重新计算 `x`、`y`、`width` 和 `height`，保证面板贴底且全宽。
- 全屏应用上呼出时使用更高层级浮窗，并保持窗口不抢占输入焦点超过必要时间。

## 17. 设计验收清单

- 主面板与设置窗口有明确区别：一个是快速工作浮层，一个是标准偏好窗口。
- 主面板固定在鼠标所在显示器底部，宽度等于显示器完整宽度，覆盖 Dock 区域，默认高度约 320 px。
- 主面板只允许通过顶部横条调整高度，禁止调宽或自由缩放。
- 剪贴板条目能在不打开预览浮层的情况下识别类型、来源、时间和内容摘要。
- 搜索、类型 chip、鼠标横向浏览、双击取用、键盘导航和预览形成连续工作流。
- 空态、加载态、错误态均有明确视觉和可恢复路径。
- 使用 macOS 系统材料与语义色，但不复刻 Paste 的品牌、文案、卡片节奏或独特视觉组合。
- 所有尺寸、间距、颜色、字体与交互细节可直接交给 macOS 工程实现。
